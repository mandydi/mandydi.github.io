类的属性

  1. 初始值:成员变量一定有默认初始值
  2. 储存空间:局部变量在栈空间
  3. 权限修饰符:局部变量没有修饰词
  4. 声明位置:局部声明在方法内，参数表;

继承/抽象/接口

abstract

  • 被abstract修饰的类成为抽象类，不可被实例化，抽象类有构造器，抽象类可以没有抽象方法，但含有抽象方法的类一定是抽象类
  • 被abstract修饰的方法为抽象方法,没有方法体,不能被private\static\synchronized等修饰，必须要以分号结尾;
public abstract void eat();
  • 若子类继承抽象类，没有重写所有的抽象方法，意味着此类中仍有抽象方法，则此类必须声明为抽象的;

Interface

被interface修饰的类为接口,可以使用接口间接实现多重继承;

接口的变量默认为public static final；

重载是一个类里面有不同的参数和函数，重写是子类与父类之间有相同名字的函数需要复写，建议从内存空间出发理解每个类有自己的储存空间

实现一定要重写接口函数，方法需要public前缀

面向接口编程的思想:接口没有构造器 可以定义接口的匿名对象

面向对象总结

面向对象的三条主线:

  1. 类及类的成分

    1. 类的设计有并列关系、关联关系、继承关系

    2. 类的成分:

      • 属性

      成员变量/局部变量

      基本数据类型/引用数据类型

      属性的声明格式:修饰符 数据类型 变量名=初始化值;

      对属性的赋值操作:默认初始化2.显式初始化3.代码块的初始化4.构造器的初始化；5.调用方法:属性进行赋值

      • 方法

      声明方法的格式:修饰符 {其他的关键字:static/abstract/final}数据类型 方法名(参数列表){//方法体}

      方法重载(overload)vs方法重写(override)

      • 构造器

      创建构造器的作用:初始化属性值;实例化类对象

      构造器也是可以重载的:多参数调用虚拟函数

    3. 代码块

      代码块的分类

      • 内部类

      成员内部类/局部内部类

    4. 类的初始化

      • 如何创建类的对象
      Person p=new Person();
      • 内存解析

      栈:局部变量 对象引用 数据引用

      堆:new的东西

      方法区:(字符串常量)

      静态域:存放类中静态的变量

  2. 面向对象的三大特性

    • 封装性:通过私有类的成员变量，通过公共的Setter和Getter方法来调用和修改

      ​ 权限修饰符:public protected 缺省 private

    • 继承性:通过一个类A继承另一个类B就能获取B的结构

    • 多态性:方法的重写和重载

      ​ 向下转型/向上转型 Student s=(Student)p;//建议向下转型前

      if(p instanceof Student)//避免出现ClassCastException的异常

  3. 其他的关键字:

    super 属性方法构造器 继承
    this 属性方法构造器 当前对象
